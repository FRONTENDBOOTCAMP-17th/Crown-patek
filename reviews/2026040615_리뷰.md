## 2. 잘한 점

### 메인 비디오 캐러셀 구현이 훌륭해요!
- `mainVideo.js`가 비디오/이미지 혼합 캐러셀을 **외부 라이브러리 없이 직접 구현**한 것이 정말 대단해요! 비디오의 `timeupdate` 이벤트로 프로그레스 바를 동기화하고, 이미지일 때는 `requestAnimationFrame`으로 타이머를 돌리는 방식이 프로답워요.
- 비디오 → 비디오, 비디오 → 이미지, 이미지 → 비디오 전환이 모두 자연스럽게 동작해요.
- `cancelAnimationFrame`으로 이전 타이머를 정리하는 것도 메모리 누수를 방지하는 좋은 습관이에요.

### 드래그 슬라이더도 직접 구현했어요!
- `mainSlide.js`에서 마우스 드래그로 슬라이드를 넘기는 기능을 라이브러리 없이 만든 것이 인상적이에요. `mousedown` → `mousemove` → `mouseup` 시퀀스를 정확히 이해하고 구현했어요.
- 드래그 거리가 짧으면 원래 자리로 돌아오는 "스냅 백(snap-back)" 동작도 잘 구현했어요.
- `cubic-bezier(0.25, 1, 0.5, 1)` 커스텀 이징으로 자연스러운 애니메이션까지 적용했어요.
- 썸네일 클릭과 드래그 슬라이드가 `goToSlide()`라는 하나의 함수로 통합되어 **응집도가 높아요**!

### 프로필 페이지 UI가 깔끔해요
- 프로필, 구매 내역, 위시리스트 3개 페이지의 디자인이 통일되어 있어요. 탭 버튼 영역, 헤더, 푸터가 일관성 있게 구성되어 있어요.
- 반응형 디자인도 잘 적용했어요 — `max-md:px-3`, `max-md:text-[10px]` 등으로 모바일 대응이 되어 있어요.
- 로그아웃 버튼을 데스크톱에서는 탭 바 옆에, 모바일에서는 하단에 배치한 것이 UX적으로 좋아요.

### CSS 변수로 디자인 토큰을 정리했어요
- `src/components/style.css`에 `--subTitle-button`, `--text-primary`, `--admin-green` 등 색상 변수를 정의한 것이 좋아요. 나중에 색상을 바꿀 때 한 곳만 수정하면 돼요.
- user 페이지와 admin 페이지의 변수를 분류해서 정리한 것도 깔끔해요.

### main.js에 null 체크를 추가했어요
- 이전 리뷰에서 지적한 `btn` null 체크를 `if (btn) { ... }` 패턴으로 정확히 추가했어요. 이제 메뉴 버튼이 없는 페이지에서도 에러가 발생하지 않아요.

### 관리자 페이지 시작이 좋아요
- `admin/components/issue-62-components.html`에서 관리자 UI 컴포넌트를 먼저 만들어보고 조합하는 접근 방식이 좋아요. 컴포넌트 단위로 개발하는 것은 좋은 습관이에요.

### 3-1. 헤더+푸터 복사가 이제 9곳으로 늘어났어요 — 결합도가 아주 높아요 (중요도: 높음)

**현재 상태:**

이전 리뷰에서 6곳이었던 헤더 복사가 프로필 3개 페이지가 추가되면서 9곳이 되었어요:

1. `index.html`
2. `src/pages/sunglasses.html`
3. `src/pages/order.html`
4. `src/components/header/header.html`
5. `src/components/login/login.html`
6. `src/components/login/register.html`
7. `src/components/profile/profile.html` ← 신규
8. `src/components/profile/buyProduct.html` ← 신규
9. `src/components/profile/wishList.html` ← 신규

그리고 **푸터**도 프로필 3개 페이지에서 모두 복사되어 있어요 (각각 ~60줄).

**왜 개선하면 좋을까? (응집도 & 결합도 관점)**

이 상태는 **결합도가 가장 높은 패턴**이에요. "복사-붙여넣기"는 코드를 공유하는 것이 아니라 **코드가 서로 묶여 있는 것**이에요. 메뉴에 "악세서리" 항목을 추가하려면 9개 파일을 모두 수정해야 하고, 하나라도 빠뜨리면 버그가 돼요.

> **쉽게 말하면:** 9명의 학생이 같은 시험지를 각자 베껴 쓴 상태예요. 시험 문제가 바뀌면 9명 모두 다시 써야 해요. 한 명이 원본을 가지고 나머지가 그걸 참조하면 훨씬 편하겠죠?

**추천:**

```javascript
// src/components/header/loadHeader.js
export async function loadHeader() {
  const res = await fetch('/src/components/header/header.html');
  const html = await res.text();
  document.querySelector('.header-placeholder').innerHTML = html;
}
```

```html
<!-- 각 페이지에서 -->
<div class="header-placeholder"></div>
<script type="module">
  import { loadHeader } from '/src/components/header/loadHeader.js';
  loadHeader();
</script>
```

푸터도 같은 방식으로 `footer.html`을 만들고 동적으로 삽입하면 **헤더는 1곳, 푸터도 1곳**에서만 관리하면 돼요.

### 3-2. 탭 버튼이 페이지를 전환하지 않아요 — 응집도가 부족해요 (중요도: 높음)

**현재 상태:**

```javascript
// src/components/profile/clickEvent.js
const tabButtons = document.querySelectorAll(".tabBtn");

tabButtons.forEach((button) => {
  button.addEventListener("click", () => {
    // CSS 클래스만 토글 — 페이지 이동은 없어요!
    tabButtons.forEach((btn) => {
      btn.classList.remove(...active);
      btn.classList.add(...inactive);
    });
    button.classList.add(...active);
    button.classList.remove(...inactive);
  });
});
```

"구매한 제품", "위시리스트", "프로필" 버튼을 클릭하면 **버튼 색상만 바뀌고** 실제로 해당 페이지로 이동하지 않아요. 3개의 별도 HTML 파일(`buyProduct.html`, `wishList.html`, `profile.html`)이 있지만 서로 연결되어 있지 않아요.

**왜 개선하면 좋을까? (응집도 관점)**

"탭 전환"이라는 기능은 "시각적 피드백" + "콘텐츠 전환" 두 가지가 함께 동작해야 해요. 지금은 시각적 피드백만 있고 콘텐츠 전환이 빠져 있어서, **기능의 응집도가 낮아요**. 사용자가 "위시리스트" 버튼을 클릭하면 당연히 위시리스트가 보일 거라고 기대하는데, 버튼 색만 바뀌면 혼란스러워요.

**추천 — 방법 A: 페이지 이동 방식**

```javascript
// src/components/profile/clickEvent.js
const tabButtons = document.querySelectorAll(".tabBtn");
const pages = {
  "구매한 제품": "/src/components/profile/buyProduct.html",
  "위시리스트": "/src/components/profile/wishList.html",
  "프로필": "/src/components/profile/profile.html",
};

tabButtons.forEach((button) => {
  button.addEventListener("click", () => {
    const page = pages[button.textContent.trim()];
    if (page) window.location.href = page;
  });
});
```

**추천 — 방법 B: 한 페이지에 콘텐츠 합치기 (더 좋은 방법)**

3개 페이지의 콘텐츠를 하나의 HTML에 넣고, 탭 클릭 시 `hidden` 클래스로 전환하면 헤더/푸터 중복도 한 번에 해결돼요!

**왜 개선하면 좋을까?**

지금은 이 스크립트들이 `index.html`에서만 로드되니까 잘 동작해요. 하지만 나중에 다른 페이지에서 실수로 이 스크립트를 로드하면 "Cannot read properties of null" 에러가 발생해요. `main.js`에서 `if (btn)` 체크를 추가한 것처럼, 여기도 같은 패턴을 적용하면 안전해요.

**추천:**

```javascript
// mainSlide.js — 맨 위에 가드 추가
const wrapper = document.querySelector(".swiper-wrapper");
if (!wrapper) {
  // 이 페이지에 슬라이더가 없으면 여기서 멈춤
  console.log("슬라이더 요소가 없습니다.");
} else {
  const slides = wrapper.querySelectorAll(".swiper-slide");
  // ... 나머지 코드
}
```

### 3-5. `mainSlide.js`에서 `WebKitCSSMatrix`는 오래된 API예요 (중요도: 중간)

**현재 상태:**

```javascript
// src/components/slide/mainSlide.js 42번째 줄
const matrix = new WebKitCSSMatrix(style.transform);
```

**왜 개선하면 좋을까?**

`WebKitCSSMatrix`는 "WebKit"이라는 이름에서 알 수 있듯이 Safari 브라우저 전용으로 만들어진 옛날 API예요. 지금은 모든 브라우저에서 `DOMMatrix`라는 **표준 API**를 지원해요. 기능은 똑같지만 이름이 표준화되었어요.

**추천:**

```javascript
// 수정 전
const matrix = new WebKitCSSMatrix(style.transform);

// 수정 후
const matrix = new DOMMatrix(style.transform);
```

### 3-6. 관리자 테이블 `<thead>`에 `<td>` 대신 `<th>`를 사용하세요 (중요도: 중간)

**현재 상태:**

```html
<!-- admin/components/issue-62-components.html 70~78번째 줄 -->
<thead>
  <tr class="text-[12px] text-(--admin-gray)">
    <td class="py-5">주문번호</td>      <!-- td를 사용 -->
    <td class="py-5">사용자 ID</td>
    <td class="py-5">수령인</td>
    <td class="py-5">금액</td>
    <td class="py-5">주문일시</td>
    <td class="py-5">작업</td>
  </tr>
</thead>
```

**왜 개선하면 좋을까?**

`<thead>` 안의 셀은 **테이블 머리글(header)**이니까 `<th>` 태그를 써야 해요. `<td>`는 **데이터 셀(data cell)**이에요. `<th>`를 쓰면 스크린 리더가 "주문번호 — 열 머리글"이라고 알려줘서 시각장애인도 표를 이해할 수 있어요. 또한 `<th>`는 기본적으로 굵은 글씨(bold)와 가운데 정렬이 적용돼요.

**추천:**

```html
<thead>
  <tr class="text-[12px] text-(--admin-gray)">
    <th class="py-5 text-left font-normal">주문번호</th>
    <th class="py-5 text-left font-normal">사용자 ID</th>
    <th class="py-5 text-left font-normal">수령인</th>
    <th class="py-5 text-left font-normal">금액</th>
    <th class="py-5 text-left font-normal">주문일시</th>
    <th class="py-5 text-left font-normal">작업</th>
  </tr>
</thead>
```

### 3-7. 관리자 페이지가 `main.js`를 로드하고 있어요 — 불필요한 결합 (중요도: 중간)

**현재 상태:**

```html
<!-- admin/components/issue-62-components.html 117번째 줄 -->
<script type="module" src="../../src/main.js"></script>
```

**왜 개선하면 좋을까? (결합도 관점)**

`main.js`는 메인 쇼핑몰 페이지의 헤더 스크롤 이벤트와 메뉴 버튼 이벤트를 처리해요. 관리자 페이지는 이 기능이 필요 없어요. 이렇게 관련 없는 코드를 로드하는 것은 **불필요한 결합**이에요.

지금은 `main.js`에 null 체크가 있어서 에러가 나지 않지만, `main.js`에 메인 페이지 전용 코드가 추가될 때마다 관리자 페이지에도 영향을 줄 수 있어요.

**추천:**

관리자 페이지 전용 `admin/main.js`를 따로 만들거나, 필요 없다면 script 태그를 제거하세요.

### 3-8. `mainVideo.js`에서 `innerText` 대신 `textContent`를 쓰면 더 좋아요 (중요도: 낮음)

**현재 상태:**

```javascript
// src/components/video/mainVideo.js 45번째 줄
titleElement.innerText = data.title;
```

**왜 개선하면 좋을까?**

`innerText`와 `textContent`는 비슷하지만 차이가 있어요:
- `textContent`는 요소의 텍스트만 단순히 바꿔요 (빠르고 안전)
- `innerText`는 화면에 보이는 텍스트 기준으로 동작하고, CSS 레이아웃을 다시 계산해요 (느림)

단순 텍스트를 설정할 때는 `textContent`가 더 효율적이에요.

**추천:**

```javascript
// 수정 전
titleElement.innerText = data.title;

// 수정 후
titleElement.textContent = data.title;
```

### 3-9. 프로필 페이지에 시맨틱 태그가 없어요 (중요도: 낮음)

<!-- profile.html — <body> 바로 아래가 전부 <div> -->

### 3-10. `src/components/style.css`에 서로 다른 역할이 섞여 있어요 — 응집도를 높이면 좋아요 (중요도: 낮음)

**현재 상태:**

```css
/* src/components/style.css */
:root{
    /* user 페이지 */
    --subTitle-button: #DFE3E8;
    --text-primary: #111111;
    /* ... */

    /* admin 페이지 */
    --admin-green: #00A63E;
    /* ... */
}

#subNav {
  cursor: grab;
}
#subNav.dragging {
  cursor: grabbing;
}
```

**왜 개선하면 좋을까? (응집도 관점)**

이 파일에 3가지 역할이 섞여 있어요:
1. **user 페이지 디자인 토큰** (색상 변수)
2. **admin 페이지 디자인 토큰** (색상 변수)
3. **특정 컴포넌트의 스타일** (#subNav 커서)

파일 하나가 여러 역할을 하면 "이 파일이 뭐하는 파일이지?" 하고 헷갈릴 수 있어요.

**추천:**

```css
/* src/components/style.css — 공용 디자인 토큰만 */
:root {
    --text-primary: #111111;
    --text-gray: #858585;
    --border: #D9D9D9;
    --bg: #F3F4F6;
}
```

관리자 전용 변수는 `admin/style.css`에, 컴포넌트 전용 스타일은 해당 컴포넌트 폴더에 두면 각 파일의 역할이 명확해져요.
