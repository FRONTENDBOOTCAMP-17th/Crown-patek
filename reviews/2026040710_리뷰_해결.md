## 3-1. search.js 이벤트 리스너 중복 등록 수정 (중요도: 높음)

### 문제

`admin/pages/components/order/search.js`의 `search()` 함수가 `renderOrderList()`에서 매번 호출되는데, 호출될 때마다 `addEventListener`로 새 리스너를 추가하고 있어요. 페이지를 넘길 때마다 리스너가 쌓여서 **메모리 누수**가 발생해요.

### 해결 방법

**Step 1.** `search.js`를 완전히 다시 작성하세요. "이벤트 등록"과 "데이터 갱신"을 분리해요.

```javascript
// 수정 전 (admin/pages/components/order/search.js 전체)
export function search(orders, input, onSearch) {
  if (!input) return;

  input.addEventListener("input", () => {
    const keyword = input.value.trim().toLowerCase();

    if (!keyword) {
      onSearch(orders);
      return;
    }

    const filteredOrders = orders.filter((item) => {
      const recipientName = item.shippingAddress.recipientName.toLowerCase() ?? "";
      const orderNumber = item.orderNumber.toLowerCase() ?? "";

      return (
        recipientName.includes(keyword) ||
        orderNumber.includes(keyword)
      );
    });

    onSearch(filteredOrders);
  });
}

// 수정 후
let currentOrders = [];

export function initSearch(input, onSearch) {
  if (!input) return;

  input.addEventListener("input", () => {
    const keyword = input.value.trim().toLowerCase();

    if (!keyword) {
      onSearch(currentOrders);
      return;
    }

    const filteredOrders = currentOrders.filter((item) => {
      const recipientName = (item.shippingAddress?.recipientName ?? "").toLowerCase();
      const orderNumber = (item.orderNumber ?? "").toLowerCase();

      return (
        recipientName.includes(keyword) ||
        orderNumber.includes(keyword)
      );
    });

    onSearch(filteredOrders);
  });
}

export function updateSearchData(orders) {
  currentOrders = orders;
}
```

**Step 2.** `index.js`에서 호출 방식을 변경하세요.

```javascript
// 수정 전 (admin/pages/components/order/index.js)
```

## 3-2. search.js null 접근 에러 수정 (중요도: 높음)

### 문제

`admin/pages/components/order/search.js` 13~14번째 줄에서 `.toLowerCase()`를 `??`보다 먼저 호출하고 있어요. `shippingAddress`나 `recipientName`이 `null`이면 에러가 발생해요.

### 해결 방법

**Step 1.** 3-1번 해결 가이드에서 이미 수정된 코드에 포함되어 있어요! 핵심은 이 부분이에요:

```javascript
// 수정 전 (search.js 13~14번째 줄) — 위험!
const recipientName = item.shippingAddress.recipientName.toLowerCase() ?? "";
const orderNumber = item.orderNumber.toLowerCase() ?? "";

// 수정 후 — 안전!
const recipientName = (item.shippingAddress?.recipientName ?? "").toLowerCase();
const orderNumber = (item.orderNumber ?? "").toLowerCase();
```

**무엇이 달라졌나요?**

| | 수정 전 | 수정 후 |
|---|---|---|
| 실행 순서 | `.toLowerCase()` → `??` | `??` → `.toLowerCase()` |
| null일 때 | 에러! | `""` → `""` (안전) |
| 핵심 변경 | - | `?.`와 괄호를 추가 |

- `?.`는 **옵셔널 체이닝(Optional Chaining)** 연산자예요. `item.shippingAddress?.recipientName`은 "shippingAddress가 있으면 recipientName을 읽고, 없으면 undefined를 반환해"라는 뜻이에요.
- `(값 ?? "")` 괄호로 감싸면 먼저 `null`/`undefined`를 빈 문자열로 바꾸고, 그다음에 `.toLowerCase()`가 실행돼요.

> **쉽게 말하면:** "선물 상자를 열어서 → 안에 있는 장난감으로 놀아라"가 아니라, "선물 상자가 있는지 확인하고 → 없으면 빈 상자를 준비하고 → 그다음 열어라"예요.

```javascript
    window.location.href = "admin-dashboard.html";
```

**Step 2.** 대시보드와 주문 페이지에 인증 확인 코드를 추가하세요.

```html

```

> **쉽게 말하면:** 놀이공원 입장 시 **팔찌(토큰)**를 채워주고, 놀이기구 탈 때마다 **팔찌를 확인**하고, 퇴장할 때 **팔찌를 반납**하는 흐름이에요.

## 3-8. calcListNum.js 인터페이스 개선 (중요도: 중간)

### 문제

`calcListNum.js`가 전체 API 응답 객체(`data`)를 받아서 내부에서 `data.data.pagination?.totalCount`를 다시 파고 있어요.

### 해결 방법

**Step 1.** `calcListNum.js`를 수정하세요.

```javascript
// 수정 전 (admin/pages/components/order/calcListNum.js 전체)
export function calcListNum(data, range, currentPage) {
    const pageSize = 20;
    const totalCount = data.data.pagination?.totalCount ?? 0;

    const start = (currentPage - 1) * pageSize + 1;
    const end = Math.min(currentPage * pageSize, totalCount);

    range.textContent = `${start} - ${end}`;
}

// 수정 후
export function calcListNum(totalCount, range, currentPage) {
    const pageSize = 20;

    const start = (currentPage - 1) * pageSize + 1;
    const end = Math.min(currentPage * pageSize, totalCount);

    range.textContent = `${start} - ${end}`;
}
```

**Step 2.** `index.js`에서 호출 부분을 수정하세요.

```javascript
// 수정 전 (index.js 43번째 줄)
calcListNum(data, range, currentPage);

// 수정 후
calcListNum(totalCount, range, currentPage);
```

`totalCount`는 이미 28번째 줄에서 꺼내놨으니까 그대로 쓰면 돼요!

> **쉽게 말하면:** "택배 상자 전체를 건네주고 안에서 찾아"가 아니라, "필요한 물건만 꺼내서 건네주기"예요. 받는 사람이 훨씬 편해요.

## 3-10. orders.html `<td>` → `<th>` 변경 (중요도: 중간)

### 문제

`admin/pages/orders.html` 57~65번째 줄의 `<thead>` 안에 `<td>`가 사용되고 있어요.

### 해결 방법

**Step 1.** `orders.html` 57~64번째 줄을 수정하세요.

```html
<!-- 수정 전 -->
<tr class="text-[12px] text-(--admin-gray)">
  <td class="py-5">주문번호</td>
  <td class="py-5">사용자 ID</td>
  <td class="py-5">수령인</td>
  <td class="py-5">금액</td>
  <td class="py-5">주문일시</td>
  <td class="py-5">작업</td>
</tr>

<!-- 수정 후 -->
<tr class="text-[12px] text-(--admin-gray)">
  <th class="py-5 text-left font-normal">주문번호</th>
  <th class="py-5 text-left font-normal">사용자 ID</th>
  <th class="py-5 text-left font-normal">수령인</th>
  <th class="py-5 text-left font-normal">금액</th>
  <th class="py-5 text-left font-normal">주문일시</th>
  <th class="py-5 text-left font-normal">작업</th>
</tr>
```

**왜 `text-left font-normal`을 추가하나요?**

`<th>`는 기본적으로 **가운데 정렬 + 굵은 글씨**예요. 현재 디자인은 왼쪽 정렬 + 보통 굵기이니까, `text-left font-normal`로 기본값을 덮어써야 기존 디자인이 유지돼요.

> **쉽게 말하면:** 표의 맨 윗줄은 "제목"이에요. HTML에서도 "이건 제목이야"라고 알려줘야 컴퓨터가 이해할 수 있어요.

## 3-11. 베스트셀러 상품 데이터 중복 해결 (중요도: 중간)

### 문제

`index.html`의 베스트셀러 섹션에서 6개 상품이 모두 같은 이름과 가격이에요.

### 해결 방법

이것은 3차 리뷰의 3-3번(신제품 슬라이드 중복)과 같은 해결 방법이에요. [이전 해결 가이드](./2026040615_리뷰_해결.md)를 참고해서, 상품 데이터를 JavaScript 배열로 만들고 동적으로 HTML을 생성하세요.

간단한 예시:

```javascript
// src/components/bestseller/bestsellerData.js
export const bestProducts = [
  { name: "벨로 M01", price: "₩269,000", image: "https://...11005074_FRONT.jpg" },
  { name: "오리가미 02", price: "₩350,000", image: "https://...11005068_FRONT.jpg" },
  // ... 6개 상품
];
```

> **쉽게 말하면:** 6명의 캐릭터를 만들 때, 한 명을 만들고 5번 복사하면 다 똑같아요. 각 캐릭터의 "설정표"를 먼저 만들고, 설정표를 보고 하나씩 그리면 다 다른 캐릭터가 돼요.

## 3-12. admin-login.html 인라인 이벤트 핸들러 제거 (중요도: 낮음)

### 문제

`admin-login.html` 21번째 줄에서 `onsubmit="handleLogin(event)"` 인라인 핸들러를 사용하고 있어요.

### 해결 방법

3-4번(admin-login.js 정리)을 방법 B로 해결하면 이 문제도 자동으로 해결돼요!

인라인 스크립트를 `admin-login.js`로 이동하면서 `addEventListener`로 바꾸고, `<form>` 태그에서 `onsubmit` 속성을 제거하면 됩니다.

```html
<!-- 수정 전 -->
<form id="loginForm" onsubmit="handleLogin(event)" novalidate>

<!-- 수정 후 -->
<form id="loginForm" novalidate>
```

> **쉽게 말하면:** HTML은 "어떻게 생겼는지(구조)"만 담당하고, JavaScript는 "어떻게 동작하는지(행동)"만 담당하면 각자 역할이 명확해져요.
