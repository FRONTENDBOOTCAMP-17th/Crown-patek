## 2. 잘한 점

### HTML 기본기
- 모든 페이지에서 `<!doctype html>`, `lang`, `charset`, `viewport` 메타태그를 빠짐없이 설정했어요. 기본기가 탄탄합니다!
- `<header>`, `<footer>`, `<nav>`, `<main>`, `<aside>`, `<figure>` 같은 시맨틱 태그를 적극적으로 활용하고 있어요. 특히 `sunglasses.html`의 `<header>`, `<main>`, `<aside>` 구분이 잘 되어 있어요.

### Tailwind CSS v4 활용
- Tailwind CSS v4의 최신 Vite 플러그인(`@tailwindcss/vite`)을 올바르게 설정해서 사용하고 있어요. CDN이 아닌 빌드 도구와 함께 쓰고 있는 것이 매우 좋습니다.
- `@layer components`를 활용해서 `.nav-bar`, `.nav-bar-list`, `.rolling-list` 같은 반복되는 스타일을 재사용 가능하게 만들었어요. 이건 Tailwind를 잘 이해하고 쓰는 거예요!
- `@theme`에서 커스텀 폰트(`--font-gentle`)와 키프레임 애니메이션(`rolling`)을 정의한 것도 Tailwind v4의 테마 시스템을 잘 활용한 거예요.
- 반응형 디자인도 `max-md:`, `md:`, `lg:`, `min-[1240px]:` 등 다양한 브레이크포인트를 사용해서 꼼꼼하게 적용했어요.

### JavaScript 구조화
- **데이터 분리가 훌륭해요!** `src/data/collection.js`와 `src/data/sunglasses.js`로 데이터를 별도 파일로 분리한 것은 정말 좋은 설계예요. 데이터가 바뀌어도 HTML이나 로직 코드를 건드릴 필요가 없어요.
- `renderNavigation.js`의 `renderNavigation()` 함수는 재사용성이 아주 뛰어나요. 컬렉션과 선글라스 양쪽에서 같은 함수를 호출하면서 `options`로 다른 동작을 주입하는 패턴이 깔끔합니다.
- `navigationEvent.js`의 드래그 & 스크롤 구현이 인상적이에요. `pointerdown → pointermove → pointerup` 이벤트 흐름, `dragThreshold`로 클릭과 드래그를 구분하는 로직, `requestAnimationFrame`으로 상태 업데이트하는 것 모두 수준 높은 구현이에요.
- `item.js`의 `createItem()` 함수에서 `aria-label="찜하기"`를 넣어 접근성을 신경 쓴 것도 좋아요.

### 인터랙션 디자인
- 스크롤에 따라 헤더 색상이 바뀌는 효과(`is-scrolled`)와 Tailwind의 `transition-all duration-300`을 조합한 것이 부드러워요.
- 롤링 배너 애니메이션(`@keyframes rolling`)이 CSS만으로 구현되어 있어서 성능도 좋고, 코드도 깔끔해요.
- 햄버거 메뉴 버튼의 X 전환 애니메이션(`rotate-45`, `-rotate-45`)도 SVG rect를 활용해서 독창적으로 구현했어요.
- `menuAnimation.js`에서 메뉴 아이템들이 순서대로 나타나는 스태거 애니메이션(`transitionDelay: index * 80ms`)이 사용자 경험을 크게 높여줘요.

### 대시보드 관리 페이지
- SPA 방식으로 탭 전환을 구현한 `navigation.js`가 잘 만들어져 있어요. `data-select` 속성으로 버튼과 페이지를 연결하고, 이벤트 위임(`e.target.closest()`)을 사용한 것이 효율적이에요.
- 사이드바의 열기/닫기 + 배경 오버레이(`wrap`) 처리도 꼼꼼해요.

## 3. 개선하면 좋을 점

### 3-1. `<ul>` 안에 `<div>`가 직접 들어가 있어요 (중요도: 높음)

**현재 상태:**

```html
<!-- index.html 10~15번째 줄 -->
<ul class="utility-bar ...">
  <div class="flex items-center max-md:hidden">   <!-- ❌ ul의 직접 자식으로 div -->
    <li class="menu-item group relative">
      ...
    </li>
  </div>
  <li class="logo-item ...">...</li>
</ul>
```

**왜 개선하면 좋을까?**
HTML 표준에서 `<ul>` 태그 안에는 `<li>` 태그만 직접 올 수 있어요. `<div>`를 직접 넣으면 브라우저가 예상치 못한 방식으로 HTML을 해석할 수 있고, 스크린 리더 같은 접근성 도구에서도 문제가 생길 수 있어요. `sunglasses.html`의 헤더에서도 같은 문제가 있어요.

**추천:** `<div>`를 `<li>`로 바꾸거나, 전체 구조를 `<nav>` + `<div>` 기반으로 재설계하면 돼요.

```html
<!-- 방법 1: div를 li로 교체 -->
<ul class="utility-bar ...">
  <li class="flex items-center max-md:hidden list-none">
    <ul class="flex">
      <li class="menu-item group relative">...</li>
    </ul>
  </li>
</ul>

<!-- 방법 2: nav 기반 구조 (추천) -->
<nav class="utility-bar ...">
  <div class="flex items-center max-md:hidden">
    <button class="nav-bar">선글라스</button>
    ...
  </div>
  <div class="logo-item ...">...</div>
</nav>
```

### 3-2. `navigationButtons.js` 파일이 존재하지 않아요 (중요도: 높음)

**현재 상태:**

```html
<!-- collection.html 27번째 줄 -->
<script type="module" src="/src/components/navigation/navigationButtons.js"></script>

<!-- collection-detail.html 70번째 줄 -->
import "../components/navigation/navigationButtons.js";
```

**왜 개선하면 좋을까?**
`/src/components/navigation/` 폴더에 `navigationButtons.js` 파일이 없어요. 실제 파일은 `collectionPage.js`인 것 같아요. 이렇게 되면 브라우저 콘솔에 **404 에러**가 뜨고, 컬렉션 페이지와 컬렉션 상세 페이지의 네비게이션 버튼이 동작하지 않아요.

**추천:** 스크립트 경로를 실제 파일명으로 수정하세요.

```html
<!-- collection.html: 수정 후 -->
<script type="module" src="/src/components/navigation/collectionPage.js"></script>

<!-- collection-detail.html: 수정 후 -->
<script type="module">
  import "../components/navigation/collectionPage.js";
  ...
</script>
```

### 3-3. `main.js`가 `sunglasses.html`에서 에러가 발생해요 (중요도: 높음)

**현재 상태:**

```javascript
// src/main.js 17~21번째 줄
const btn = document.getElementById("menu-btn");
const line1 = document.getElementById("line1");
const line2 = document.getElementById("line2");

btn.addEventListener("click", () => { ... });  // ❌ btn이 null이면 에러!
```

```html
<!-- index.html: id="menu-btn" ✅ -->
<button id="menu-btn" class="...">

<!-- sunglasses.html: id="menuButton" ❌ 이름이 달라요! -->
<button id="menuButton" class="...">
```

**왜 개선하면 좋을까?**
`main.js`는 `index.html`과 `sunglasses.html` 둘 다에서 로드되는데, `sunglasses.html`에는 `id="menu-btn"` 요소가 없어요 (대신 `id="menuButton"`). 그래서 `btn`이 `null`이 되고, `null.addEventListener`를 호출하면서 **TypeError**가 발생해요. 이러면 `main.js`의 나머지 코드도 실행이 안 돼요.

**추천:** null 체크를 추가하거나, 모든 페이지에서 같은 id를 사용하세요.

```javascript
// src/main.js — null 체크 추가
const btn = document.getElementById("menu-btn");

if (btn) {
  const line1 = document.getElementById("line1");
  const line2 = document.getElementById("line2");

  btn.addEventListener("click", () => {
    line1.classList.toggle("translate-y-[2.5px]");
    line1.classList.toggle("rotate-45");
    line2.classList.toggle("-translate-y-[2.5px]");
    line2.classList.toggle("-rotate-45");
    header.classList.toggle("is-active");
  });
}
```

### 3-4. `menuAnimation.js`에서 `header` 변수가 정의되지 않았어요 (중요도: 높음)

**현재 상태:**

```javascript
// src/components/mobilemenu/menuAnimation.js 73번째 줄
button.addEventListener("click", () => {
  line1.classList.toggle("translate-y-[2.5px]");
  line1.classList.toggle("rotate-45");
  line2.classList.toggle("-translate-y-[2.5px]");
  line2.classList.toggle("-rotate-45");

  header.classList.toggle("is-active");  // ❌ header가 어디서도 선언되지 않았어요!
});
```

**왜 개선하면 좋을까?**
JavaScript 모듈은 각각 독립된 범위(scope)를 가져요. `main.js`에서 `header`를 선언했다 해도 `menuAnimation.js`에서는 접근할 수 없어요. 이 코드가 실행되면 **ReferenceError: header is not defined** 에러가 발생해요.

**추천:** `menuAnimation.js` 안에서 `header`를 직접 찾아오세요.

```javascript
// menuAnimation.js 상단에 추가
const header = document.querySelector(".utility-bar");

// 또는 사용하는 곳에서 직접 조회
button.addEventListener("click", () => {
  // ...
  const header = document.querySelector(".utility-bar");
  if (header) {
    header.classList.toggle("is-active");
  }
});
```

### 3-5. 헤더 HTML이 페이지마다 복사-붙여넣기 되어 있어요 (중요도: 중간)

**현재 상태:**
`index.html`의 헤더 (약 370줄)와 `sunglasses.html`의 헤더 (약 310줄)가 거의 동일한 HTML인데 별도로 작성되어 있어요. 로고 SVG만 해도 수십 줄이 중복돼요.

**왜 개선하면 좋을까?**
메뉴 항목을 하나 추가하거나 로고를 바꿀 때 여러 파일을 모두 수정해야 해요. 하나라도 빠뜨리면 페이지마다 다른 메뉴가 보이게 돼요. 이미 `sunglasses.html`의 `id`가 `index.html`과 다른 것도 이런 중복 때문에 생긴 문제예요.

**추천:** 당장은 어려울 수 있지만, 나중에 JavaScript로 헤더를 동적으로 렌더링하는 함수를 만들면 한 곳에서만 관리할 수 있어요. `renderNavigation.js`처럼 `renderHeader.js`를 만드는 거예요!

### 3-6. Vite 빌드 설정에 서브 페이지가 포함되지 않아요 (중요도: 중간)

**현재 상태:**

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
  // ❌ build.rollupOptions.input 설정이 없어요
})
```

**왜 개선하면 좋을까?**
Vite는 기본적으로 프로젝트 루트의 `index.html`만 빌드해요. `src/pages/` 안의 HTML 파일들(`sunglasses.html`, `collection.html` 등)은 `vite build` 했을 때 포함되지 않아서, 배포(deploy)하면 해당 페이지에 접근할 수 없어요. 개발 서버(`vite dev`)에서는 동작하지만 실제 배포에서는 안 돼요.

**추천:** `vite.config.js`에 멀티 페이지 설정을 추가하세요.

```javascript
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'
import { resolve } from 'path'

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        sunglasses: resolve(__dirname, 'src/pages/sunglasses.html'),
        collection: resolve(__dirname, 'src/pages/collection.html'),
        collectionDetail: resolve(__dirname, 'src/pages/collection-detail.html'),
        productDetail: resolve(__dirname, 'src/pages/product-detail.html'),
        dashboard: resolve(__dirname, 'src/components/dashboard/dashboard.html'),
      },
    },
  },
})
```

### 3-7. 링크의 `href=""`가 현재 페이지를 새로고침해요 (중요도: 중간)

**현재 상태:**

```html
<!-- index.html, sunglasses.html 등 여러 곳 -->
<a href="">전체보기</a>
<a href="">2026 컬렉션</a>
<a href="">서킷 컬렉션</a>
```

**왜 개선하면 좋을까?**
`href=""`는 "현재 페이지"를 의미해요. 클릭하면 현재 페이지가 새로고침돼요. 아직 연결할 페이지가 없다면 `href="#"`을 쓰는 게 더 안전하고, 가장 좋은 방법은 `<a>` 대신 `<button>`을 쓰거나 실제 경로를 넣는 거예요.

**추천:**
```html
<!-- 아직 페이지가 없을 때 -->
<a href="#">전체보기</a>

<!-- 또는 링크가 아닌 텍스트라면 -->
<span>전체보기</span>
```

### 3-8. `<img>` 태그의 `alt` 속성이 비어 있어요 (중요도: 중간)

**현재 상태:**

```html
<!-- sunglasses.html, submenu.html 등 -->
<img src="https://gm-prd-resource..." alt="" />
```

**왜 개선하면 좋을까?**
`alt=""`는 "이 이미지는 장식용"이라는 의미예요. 하지만 상품 이미지는 장식이 아니라 콘텐츠의 핵심이에요! 시각 장애인이 스크린 리더를 사용할 때, 상품명이 `alt`에 없으면 어떤 상품인지 알 수 없어요. 검색엔진 최적화(SEO)에도 도움이 돼요.

**추천:**
```html
<img src="..." alt="디즈니 F1® - 윙 4 R13 선글라스" />
```

### 3-9. `index.html`에 `<main>` 태그가 없어요 (중요도: 중간)

**현재 상태:**

```html
<!-- index.html -->
<body>
  <div class="header ...">...</div>
  <div class="main-video">           <!-- ❌ div가 아니라 main이어야 해요 -->
    <video ...>...</video>
    ...
    <footer>푸터</footer>             <!-- ❌ main-video 안에 footer가 있어요 -->
  </div>
</body>
```

**왜 개선하면 좋을까?**
`<main>` 태그는 "이 페이지의 핵심 콘텐츠가 여기 있어요"라고 알려주는 역할이에요. 스크린 리더 사용자가 바로 본문 내용으로 이동할 수 있게 해줘요. 또한 `<footer>`가 `<div class="main-video">` 안에 들어있는 것도 구조적으로 어색해요.

**추천:**
```html
<body>
  <header class="header ...">...</header>
  <main class="main-video">
    <video ...>...</video>
    ...
  </main>
  <footer>...</footer>
</body>
```

### 3-10. `class` 속성이 두 번 선언되어 있어요 (중요도: 중간)

**현재 상태:**

```html
<!-- sunglasses.html 333번째 줄 -->
<a class="menu-animate w-max" href="/" class="">로그인 / 계정 생성</a>
<!--                                   ^^^^^^^  ❌ class가 두 번! -->
```

**왜 개선하면 좋을까?**
HTML에서 같은 속성을 두 번 쓰면 브라우저마다 동작이 달라질 수 있어요. 보통 첫 번째 `class`만 인식하고, 두 번째는 무시돼요. 의도하지 않은 스타일 문제가 생길 수 있어요.

**추천:** 두 번째 `class=""`를 삭제하면 돼요.
```html
<a class="menu-animate w-max" href="/">로그인 / 계정 생성</a>
```

### 3-11. 스크롤 임계값이 매직 넘버예요 (중요도: 낮음)

**현재 상태:**

```javascript
// src/main.js 7번째 줄
if (window.scrollY > 785.33) {
```

**왜 개선하면 좋을까?**
`785.33`이 무엇을 의미하는지 코드만 봐서는 알 수 없어요. 비디오 높이인 것 같은데, 화면 크기나 콘텐츠가 바뀌면 이 숫자도 같이 바꿔야 해요. 이런 것을 **매직 넘버(magic number)**라고 부르는데, 나중에 유지보수하기 어려워져요.

**추천:** 상수로 이름을 붙이거나, 실제 요소의 높이를 계산해서 사용하세요.
```javascript
// 방법 1: 의미 있는 이름 붙이기
const VIDEO_HEIGHT = 785.33;

if (window.scrollY > VIDEO_HEIGHT) { ... }

// 방법 2: 동적으로 계산 (더 좋은 방법)
const video = document.querySelector(".main-video video");
const videoHeight = video?.offsetHeight ?? 785;

if (window.scrollY > videoHeight) { ... }
```

### 3-12. 코드 포맷팅 설정이 없어요 (중요도: 낮음)

**현재 상태:**
프로젝트에 `.prettierrc`, `.eslintrc` 같은 코드 스타일 설정 파일이 없어요.

**왜 개선하면 좋을까?**
팀 프로젝트에서 각자 다른 포맷팅을 쓰면 코드 리뷰 때 불필요한 변경이 많아지고, Git 충돌도 자주 생겨요. 자동 포맷팅을 설정해두면 코드 스타일로 고민할 필요가 없어져요.

**추천:** 간단한 `.prettierrc` 파일을 추가하세요.
```json
{
  "semi": true,
  "singleQuote": false,
  "tabWidth": 2,
  "trailingComma": "all"
}
```

### 3-13. `index.html`의 콘텐츠 영역이 반복되는 플레이스홀더예요 (중요도: 낮음)

**현재 상태:**

```html
<!-- index.html 400~487번째 줄 — 같은 내용이 20번 반복 -->
<div><h2>새롭게 선보이는 젠틀몬스터 신제품</h2><span>더보기</span></div>
<div><h2>TOP 20 베스트셀러</h2><span>더보기</span></div>
<!-- ... 위 두 개가 10번 반복 ... -->
<footer>푸터</footer>
```

**왜 개선하면 좋을까?**
스크롤 테스트용 더미 콘텐츠 같은데, 이대로 두면 실제 콘텐츠로 교체하기 어렵고, `<footer>푸터</footer>`도 실제 콘텐츠가 아닌 플레이스홀더예요. 이미 `src/components/footer/footer.html`에 완성된 푸터가 있으니 활용하면 좋아요.

**추천:** 반복되는 블록을 실제 콘텐츠로 교체하거나, 스크롤 테스트가 목적이라면 주석으로 의도를 남겨두세요.

### 3-14. `<select>` 요소에 `<label>`이 연결되어 있지 않아요 (중요도: 낮음)

**현재 상태:**

```html
<!-- dashboard.html -->
<select class="w-full p-3" name="" id="">
  <option value="date">SORT BY DATE</option>
  ...
</select>
```

**왜 개선하면 좋을까?**
`<label>`이 없으면 스크린 리더 사용자가 이 선택 상자가 무엇인지 알 수 없어요. `name`과 `id`도 비어있어요.

**추천:**
```html
<label for="sort-select" class="sr-only">정렬 기준</label>
<select id="sort-select" name="sort" class="w-full p-3">
  <option value="date">SORT BY DATE</option>
  ...
</select>
```
