## 3-1. 헤더+푸터 복사가 이제 9곳으로 늘어났어요 (중요도: 높음)

### 문제

헤더 HTML이 9개 파일에 복사되어 있고, 푸터도 프로필 3개 페이지에서 중복돼요. 이전 리뷰에서 계속 지적되어 온 가장 큰 문제예요.

### 해결 방법

**Step 1.** `src/components/header/header.html`에 이미 헤더 HTML이 분리되어 있어요. 이 파일을 "원본"으로 삼으세요. 푸터도 별도 파일로 만들어요.

```html
<!-- src/components/footer/footer.html (새로 만들기) -->
<footer>
  <div class="lg:flex justify-between items-center">
    <ul class="px-6 text-[12px] mt-21.5 mb-15 md:mb-14 md:mt-16.5 lg:mb-4.25 lg:mt-21 flex flex-col lg:flex-row lg:px-15 gap-4">
      <li><a href="">문의하기</a></li>
      <li><a href="">고객 서비스</a></li>
      <li><a href="">스토어 찾기</a></li>
      <li><a href="">법적 고지</a></li>
      <li><a href="">개인정보 처리방침</a></li>
      <li><a href="">구독하기</a></li>
      <!-- ... 나머지 푸터 내용 ... -->
    </ul>
    <!-- ... -->
  </div>
</footer>
```

**Step 2.** 헤더와 푸터를 로드하는 JavaScript 함수를 만들어요.

```javascript
// src/utils/loadComponent.js (새로 만들기)
export async function loadComponent(selector, htmlPath) {
  const placeholder = document.querySelector(selector);
  if (!placeholder) return;

  const response = await fetch(htmlPath);
  const html = await response.text();
  placeholder.innerHTML = html;
}
```

**Step 3.** 각 페이지의 헤더/푸터를 플레이스홀더로 교체해요.

```html
<!-- 수정 전 (profile.html) — 360줄의 헤더 HTML -->
<div class="header relative">
  <ul class="utility-bar ...">
    <!-- ... 엄청나게 긴 헤더 코드 ... -->
  </ul>
</div>

<!-- 수정 후 — 딱 1줄 -->
<div id="header-root"></div>
```

```html
<!-- 수정 전 (profile.html) — 60줄의 푸터 HTML -->
<footer>
  <!-- ... 긴 푸터 코드 ... -->
</footer>

<!-- 수정 후 — 딱 1줄 -->
<div id="footer-root"></div>
```

**Step 4.** 각 페이지의 script에서 로드 함수를 호출해요.

```javascript
// 각 페이지의 script에 추가
import { loadComponent } from '/src/utils/loadComponent.js';

loadComponent('#header-root', '/src/components/header/header.html');
loadComponent('#footer-root', '/src/components/footer/footer.html');
```

> **쉽게 말하면:** 편지 내용을 9번 베껴 쓰는 대신, 원본 편지 1개를 복사기에 넣고 "이 편지를 붙여주세요"라고 하는 거예요. 원본만 바꾸면 모든 곳에 반영돼요.

## 3-2. 탭 버튼이 페이지를 전환하지 않아요 (중요도: 높음)

### 문제

`src/components/profile/clickEvent.js`의 탭 버튼이 CSS 클래스만 토글하고, 실제로 구매 내역/위시리스트/프로필 페이지로 이동하지 않아요.

### 해결 방법 — 프로필 3개 페이지를 하나로 합치기

이 방법이 3-1번 (헤더/푸터 중복)도 동시에 해결해줘요!

**Step 1.** `profile.html` 하나에 3개 탭의 콘텐츠를 모두 넣어요.

```html
<!-- src/components/profile/profile.html -->
<!-- 탭 버튼 영역 (기존과 동일) -->
<div class="buttonMenu px-15 flex justify-between max-md:px-3">
  <div class="px-[3.5px]">
    <button class="tabBtn mr-2.5 ..." data-tab="buyProduct">구매한 제품</button>
    <button class="tabBtn mr-2.5 ..." data-tab="wishList">위시리스트</button>
    <button class="tabBtn mr-2.5 ..." data-tab="profile">프로필</button>
  </div>
  <button class="logout max-md:hidden">로그아웃</button>
</div>

<!-- 구매 내역 탭 (처음에 보임) -->
<div id="tab-buyProduct" class="tab-content">
  <div class="flex flex-col items-center mt-15 max-md:mt-0">
    <h2 class="text-[17px] ...">최근 구매 내역</h2>
    <!-- buyProduct.html의 콘텐츠 영역만 여기에 -->
  </div>

<!-- 위시리스트 탭 (처음에 숨김) -->

  <div class="flex flex-col items-center mt-15 max-md:mt-0">
    <h2 class="text-[17px] ...">위시리스트</h2>

  </div>

  <div class="flex flex-col items-center mt-15 max-md:mt-0">
    <h2 class="text-[17px] ...">프로필</h2>
    <!-- profile.html의 프로필 콘텐츠만 여기에 -->
  </div>

```

**Step 2.** `clickEvent.js`를 탭 전환 기능으로 업데이트해요.

```javascript
// 수정 전 (clickEvent.js)
const tabButtons = document.querySelectorAll(".tabBtn");
const active = ["bg-[#dfe3e8]", "text-black", "border-transparent"];
const inactive = ["text-[#858585]", "border-[#858585]", "border"];

tabButtons.forEach((button) => {
  button.addEventListener("click", () => {
    tabButtons.forEach((btn) => {
      btn.classList.remove(...active);
      btn.classList.add(...inactive);
    });
    button.classList.add(...active);
    button.classList.remove(...inactive);
  });
});

// 수정 후 (clickEvent.js)
const tabButtons = document.querySelectorAll(".tabBtn");
const tabContents = document.querySelectorAll(".tab-content");
const active = ["bg-[#dfe3e8]", "text-black", "border-transparent"];
const inactive = ["text-[#858585]", "border-[#858585]", "border"];

tabButtons.forEach((button) => {
  button.addEventListener("click", () => {
    // 1. 모든 버튼을 비활성 상태로
    tabButtons.forEach((btn) => {
      btn.classList.remove(...active);
      btn.classList.add(...inactive);
    });
    // 2. 클릭한 버튼을 활성 상태로
    button.classList.add(...active);
    button.classList.remove(...inactive);

    // 3. 모든 탭 콘텐츠 숨기기
    tabContents.forEach((content) => {
      content.classList.add("hidden");
    });
    // 4. 해당 탭 콘텐츠 보이기
    const tabId = button.getAttribute("data-tab");
    const targetTab = document.getElementById(`tab-${tabId}`);
    if (targetTab) {
      targetTab.classList.remove("hidden");
    }
  });
});
```

**Step 3.** `buyProduct.html`과 `wishList.html`은 삭제해도 돼요 (콘텐츠가 profile.html에 합쳐졌으니까).

> **쉽게 말하면:** 책의 "목차", "1장", "2장"을 각각 별도의 책으로 만든 상태예요. 한 권의 책 안에서 탭(북마크)을 넘기듯이 바꾸는 게 훨씬 자연스러워요.

## 3-5. `WebKitCSSMatrix` → `DOMMatrix` (중요도: 중간)

### 문제

`src/components/slide/mainSlide.js` 42번째 줄에서 오래된 `WebKitCSSMatrix` API를 사용하고 있어요.

### 해결 방법

**Step 1.** 딱 한 줄만 바꾸면 돼요.

```javascript
// 수정 전 (mainSlide.js 42번째 줄)
const matrix = new WebKitCSSMatrix(style.transform);

// 수정 후
const matrix = new DOMMatrix(style.transform);
```

나머지 코드는 전혀 바꿀 필요 없어요. `DOMMatrix`도 `.m41` 속성을 똑같이 가지고 있거든요.

> **쉽게 말하면:** "구 서울역" 대신 "서울역"이라고 부르는 것과 같아요. 같은 장소인데 공식 이름이 바뀐 거예요.

## 3-6. `<thead>`에 `<th>` 사용하기 (중요도: 중간)

### 문제

`admin/components/issue-62-components.html` 71~78번째 줄에서 테이블 헤더에 `<td>`를 사용하고 있어요.

### 해결 방법

**Step 1.** `<td>`를 `<th>`로 바꾸고, 왼쪽 정렬과 일반 두께를 적용해요.

```html
<!-- 수정 전 (issue-62-components.html 70~78번째 줄) -->
<thead>
  <tr class="text-[12px] text-(--admin-gray)">
    <td class="py-5">주문번호</td>
    <td class="py-5">사용자 ID</td>
    <td class="py-5">수령인</td>
    <td class="py-5">금액</td>
    <td class="py-5">주문일시</td>
    <td class="py-5">작업</td>
  </tr>
</thead>

<!-- 수정 후 -->
<thead>
  <tr class="text-[12px] text-(--admin-gray)">
    <th class="py-5 text-left font-normal">주문번호</th>
    <th class="py-5 text-left font-normal">사용자 ID</th>
    <th class="py-5 text-left font-normal">수령인</th>
    <th class="py-5 text-left font-normal">금액</th>
    <th class="py-5 text-left font-normal">주문일시</th>
    <th class="py-5 text-left font-normal">작업</th>
  </tr>
</thead>
```

`text-left`는 `<th>`의 기본 가운데 정렬을 왼쪽으로 바꿔주고, `font-normal`은 기본 굵은 글씨를 일반으로 바꿔줘요. 이렇게 하면 기존과 시각적으로 동일하면서 의미적으로 올바른 HTML이 돼요.

> **쉽게 말하면:** 표의 맨 윗줄은 "제목 줄"이니까 제목 전용 태그(`<th>`)를 쓰는 거예요. `<td>`는 "내용 칸"이에요.
