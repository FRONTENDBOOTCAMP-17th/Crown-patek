## 3-1. `<ul>` 안에 `<div>`가 직접 들어가 있어요 (중요도: 높음)

### 문제

`index.html` 14번째 줄, `sunglasses.html` 25번째 줄에서 `<ul class="utility-bar">` 안에 `<div>`가 `<li>`의 형제로 직접 들어가 있어서 HTML 표준에 맞지 않아요.

### 해결 방법

**Step 1.** `index.html`에서 `<ul>` 안의 `<div>`를 `<li>`로 바꾸세요.

```html
<!-- 수정 전 (index.html 14번째 줄) -->
<div class="flex items-center max-md:hidden">

<!-- 수정 후 -->
<li class="flex items-center max-md:hidden list-none">
```

닫는 태그도 함께 바꿔야 해요:
```html
<!-- 수정 전 (index.html 155번째 줄) -->
</div>

<!-- 수정 후 -->
</li>
```

**Step 2.** `sunglasses.html`도 같은 방식으로 수정하세요. 25번째 줄의 `<div class="flex items-center max-md:hidden">`을 `<li class="flex items-center max-md:hidden list-none">`으로 변경하고, 158번째 줄의 `</div>`도 `</li>`로 변경하세요.

> **쉽게 말하면:** `<ul>`은 "목록"이라서 안에는 "항목(`<li>`)"만 넣을 수 있어요. `<div>` 대신 `<li>`로 감싸주면 돼요.

## 3-3. `main.js`가 `sunglasses.html`에서 에러가 발생해요 (중요도: 높음)

### 문제

`src/main.js` 17번째 줄에서 `document.getElementById("menu-btn")`으로 버튼을 찾는데, `sunglasses.html`에서는 이 버튼의 id가 `menuButton`이에요. 결과적으로 `btn`이 `null`이 되어 21번째 줄 `btn.addEventListener`에서 에러가 발생해요.

### 해결 방법

**Step 1.** `src/main.js`에 null 체크를 추가하세요.

```javascript
// 수정 전 (src/main.js 17~29번째 줄)
const btn = document.getElementById("menu-btn");
const line1 = document.getElementById("line1");
const line2 = document.getElementById("line2");

btn.addEventListener("click", () => {
  line1.classList.toggle("translate-y-[2.5px]");
  line1.classList.toggle("rotate-45");

  line2.classList.toggle("-translate-y-[2.5px]");
  line2.classList.toggle("-rotate-45");

  header.classList.toggle("is-active");
});

// 수정 후
const btn = document.getElementById("menu-btn");

if (btn) {
  const line1 = document.getElementById("line1");
  const line2 = document.getElementById("line2");

  btn.addEventListener("click", () => {
    line1.classList.toggle("translate-y-[2.5px]");
    line1.classList.toggle("rotate-45");

    line2.classList.toggle("-translate-y-[2.5px]");
    line2.classList.toggle("-rotate-45");

    header.classList.toggle("is-active");
  });
}
```

> **쉽게 말하면:** "친구가 있으면 같이 놀고, 없으면 넘어가자"라는 안전장치를 넣는 거예요. `if (btn)`이 "버튼이 있을 때만 실행해"라는 뜻이에요.

## 3-4. `menuAnimation.js`에서 `header` 변수가 정의되지 않았어요 (중요도: 높음)

### 문제

`src/components/mobilemenu/menuAnimation.js` 73번째 줄에서 `header.classList.toggle("is-active")`를 호출하지만, 이 파일 안에서 `header` 변수가 어디서도 선언되지 않아서 **ReferenceError**가 발생해요.

### 해결 방법

**Step 1.** `menuAnimation.js` 파일 상단에 `header` 변수를 추가하세요.

```javascript
// 수정 전 (menuAnimation.js 1번째 줄부터)
const button = document.getElementById("menuButton");
const menu = document.getElementById("mobileMenu");
const menuItems = menu.querySelectorAll(".menu-animate");

// 수정 후
const button = document.getElementById("menuButton");
const menu = document.getElementById("mobileMenu");
const menuItems = menu.querySelectorAll(".menu-animate");
const header = document.querySelector(".utility-bar");
```

**Step 2.** 73번째 줄 근처에도 null 체크를 추가하면 더 안전해요.

```javascript
// 수정 전 (menuAnimation.js 66~74번째 줄)
button.addEventListener("click", () => {
  line1.classList.toggle("translate-y-[2.5px]");
  line1.classList.toggle("rotate-45");

  line2.classList.toggle("-translate-y-[2.5px]");
  line2.classList.toggle("-rotate-45");

  header.classList.toggle("is-active");
});

// 수정 후
button.addEventListener("click", () => {
  line1.classList.toggle("translate-y-[2.5px]");
  line1.classList.toggle("rotate-45");

  line2.classList.toggle("-translate-y-[2.5px]");
  line2.classList.toggle("-rotate-45");

  if (header) {
    header.classList.toggle("is-active");
  }
});
```

> **쉽게 말하면:** `header`라는 변수를 파일 윗부분에서 "이 친구 알아?"하고 소개해주는 거예요. 소개 없이 갑자기 이름을 부르면 JavaScript가 "누구?"하고 에러를 던져요.

## 3-5. 헤더 HTML이 페이지마다 복사-붙여넣기 되어 있어요 (중요도: 중간)

### 문제

`index.html`과 `sunglasses.html`에 거의 동일한 헤더 HTML이 각각 300줄 이상 중복되어 있어요.

### 해결 방법

이건 지금 당장 고치기보다는, 프로젝트가 더 발전했을 때 도전하면 좋아요.

**Step 1.** `src/components/header/` 폴더를 만들고 `renderHeader.js` 파일을 만드세요.

```javascript
// src/components/header/renderHeader.js
export function renderHeader(container, options = {}) {
  const { theme = "dark" } = options;
  // dark: 메인 페이지 (흰 글씨, 투명 배경)
  // light: 서브 페이지 (검정 글씨, 밝은 배경)

  container.innerHTML = `
    <ul class="utility-bar ...">
      <!-- 여기에 공통 헤더 HTML -->
    </ul>
  `;
}
```

**Step 2.** 각 HTML 파일에서 헤더를 이 함수로 교체하세요.

```html
<div id="header"></div>
<script type="module">
  import { renderHeader } from '/src/components/header/renderHeader.js';
  renderHeader(document.getElementById('header'), { theme: 'dark' });
</script>
```

> **쉽게 말하면:** 같은 내용의 편지를 10명에게 보내야 할 때, 10번 손으로 쓰는 대신 원본 하나를 만들어서 복사하는 거예요. 원본만 고치면 전부 바뀌니까요!

## 3-6. Vite 빌드 설정에 서브 페이지가 포함되지 않아요 (중요도: 중간)

### 문제

`vite.config.js`에 `build.rollupOptions.input` 설정이 없어서, `vite build` 시 `src/pages/` 안의 HTML 파일들이 빌드 결과에 포함되지 않아요.

### 해결 방법

**Step 1.** `vite.config.js`를 아래와 같이 수정하세요.

```javascript
// 수정 전 (vite.config.js)
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
})

// 수정 후
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'
import { resolve } from 'path'

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        sunglasses: resolve(__dirname, 'src/pages/sunglasses.html'),
        collection: resolve(__dirname, 'src/pages/collection.html'),
        collectionDetail: resolve(__dirname, 'src/pages/collection-detail.html'),
        productDetail: resolve(__dirname, 'src/pages/product-detail.html'),
        dashboard: resolve(__dirname, 'src/components/dashboard/dashboard.html'),
      },
    },
  },
})
```

**Step 2.** 수정 후 `npm run build`를 실행해서 `dist/` 폴더에 모든 페이지가 포함되는지 확인하세요.

> **쉽게 말하면:** Vite에게 "이 페이지들도 같이 포장해줘"라고 목록을 알려주는 거예요. 안 알려주면 메인 페이지만 포장해요.

## 3-7. 링크의 `href=""`가 현재 페이지를 새로고침해요 (중요도: 중간)

### 문제

`index.html`, `sunglasses.html`, `footer.html` 등 여러 파일에서 `<a href="">`가 사용되고 있어요.

### 해결 방법

**Step 1.** 아직 연결할 페이지가 없는 링크는 `href="#"`으로 변경하세요.

```html
<!-- 수정 전 -->
<a href="">전체보기</a>
<a href="">2026 컬렉션</a>

<!-- 수정 후 -->
<a href="#">전체보기</a>
<a href="#">2026 컬렉션</a>
```

나중에 실제 페이지가 만들어지면 `#`을 실제 경로로 바꾸면 돼요.

> **쉽게 말하면:** `href=""`는 "지금 있는 곳으로 다시 가"라는 뜻이라 페이지가 새로고침돼요. `href="#"`은 "아무 데도 안 가"라는 뜻이라 새로고침이 안 돼요.

## 3-11. 스크롤 임계값이 매직 넘버예요 (중요도: 낮음)

### 문제

`src/main.js` 7번째 줄의 `785.33`이 무엇을 의미하는지 코드만으로는 알 수 없어요.

### 해결 방법

**Step 1.** 상수에 의미 있는 이름을 붙이세요.

```javascript
// 수정 전 (src/main.js 6~11번째 줄)
const handleScroll = () => {
  if (window.scrollY > 785.33) {
    header.classList.add("is-scrolled");
  } else {
    header.classList.remove("is-scrolled");
  }
};

// 수정 후 — 방법 1: 상수 이름 붙이기
const MAIN_VIDEO_HEIGHT = 785;

const handleScroll = () => {
  if (window.scrollY > MAIN_VIDEO_HEIGHT) {
    header.classList.add("is-scrolled");
  } else {
    header.classList.remove("is-scrolled");
  }
};

// 수정 후 — 방법 2: 동적 계산 (더 좋아요)
const video = document.querySelector(".main-video video");

const handleScroll = () => {
  const threshold = video ? video.offsetHeight : 785;
  if (window.scrollY > threshold) {
    header.classList.add("is-scrolled");
  } else {
    header.classList.remove("is-scrolled");
  }
};
```

> **쉽게 말하면:** "785.33"이라는 숫자만 보면 아무도 이게 뭔지 몰라요. "비디오 높이"라고 이름을 붙여주면 나중에 봐도 바로 이해할 수 있어요.

## 3-14. `<select>` 요소에 `<label>`이 연결되어 있지 않아요 (중요도: 낮음)

### 문제

`dashboard.html`의 `<select>` 요소에 `<label>`이 없고, `name`과 `id`도 비어있어요.

### 해결 방법

**Step 1.** `<select>` 위에 `<label>`을 추가하세요.

```html
<!-- 수정 전 (dashboard.html) -->
<select class="w-full p-3" name="" id="">
  <option value="date">SORT BY DATE</option>
  <option value="name">SORT BY NAME</option>
  <option value="points">SORT BY POINTS</option>

  <option value="points">SORT BY POINTS</option>
</select>
```

`sr-only`는 Tailwind에서 "화면에는 안 보이지만 스크린 리더에는 읽히는" 클래스예요.

> **쉽게 말하면:** 눈을 감고 이 페이지를 쓴다고 생각해보세요. "뭘 선택하는 건지"를 `<label>`이 알려줘요.
