## 3-2. 사용자 이름 합치기에 연산자 우선순위 버그가 있어요 (중요도: 높음)

### 문제

`admin/pages/components/user/renderRows.js` 17번째 줄에서, `+` 연산자가 `??` 연산자보다 먼저 실행되어 `lastName`이 `undefined`일 때 `"undefined홍길동"` 같은 문자열이 표시됩니다.

### 해결 방법

**Step 1.** `admin/pages/components/user/renderRows.js` 파일을 열고 17번째 줄을 수정하세요.

```javascript
// 수정 전 (admin/pages/components/user/renderRows.js 17번째 줄)
tdId.textContent = item.lastName + item.firstName ?? "-";

// 수정 후
tdId.textContent = (item.lastName ?? "") + (item.firstName ?? "") || "-";
```

**왜 이렇게 고치나요?**

JavaScript의 연산자 우선순위를 이해해야 해요:

```
연산자 우선순위 (높은 것이 먼저 실행):
1. () 괄호            — 가장 먼저
2. + 더하기/문자열 연결  — 두 번째
3. ?? null 병합        — 세 번째
4. || OR              — 네 번째
```

원래 코드 `item.lastName + item.firstName ?? "-"`는 이렇게 실행돼요:

```
Step 1: item.lastName + item.firstName
        = undefined + "홍길동"
        = "undefined홍길동"    ← 문자열이 됨!

Step 2: "undefined홍길동" ?? "-"
        = "undefined홍길동"    ← null이 아니니까 그대로!
```

수정된 코드 `(item.lastName ?? "") + (item.firstName ?? "") || "-"`는 이렇게 실행돼요:

```
Step 1: (item.lastName ?? "")
        = (undefined ?? "")
        = ""                   ← 빈 문자열

Step 2: (item.firstName ?? "")
        = ("홍길동" ?? "")
        = "홍길동"

Step 3: "" + "홍길동"
        = "홍길동"             ← 정상!

Step 4: "홍길동" || "-"
        = "홍길동"             ← 빈 문자열이 아니니까 그대로!
```

만약 둘 다 없으면:

```
Step 3: "" + ""
        = ""                   ← 빈 문자열

Step 4: "" || "-"
        = "-"                  ← 빈 문자열은 falsy이므로 "-" 표시!
```

> **쉽게 말하면:** 수학에서 `2 + 3 × 4`가 `2 + 12 = 14`이지 `5 × 4 = 20`이 아닌 것처럼, 코드에서도 연산자 순서가 중요해요. 확실하지 않을 때는 **괄호로 감싸는** 것이 안전해요.

## 3-3. 주문 검색에서 null 값이 오면 에러가 나요 (중요도: 높음)

### 문제

`admin/pages/components/order/search.js` 13~14번째 줄에서, `recipientName`이나 `orderNumber`가 `null`/`undefined`이면 `.toLowerCase()`를 호출하는 순간 `TypeError`가 발생합니다. `??` 연산자는 `.toLowerCase()` 이후에 실행되기 때문에 에러를 방지하지 못합니다.

### 해결 방법

**Step 1.** `admin/pages/components/order/search.js` 파일을 열고 12~14번째 줄을 수정하세요.

```javascript
// 수정 전 (admin/pages/components/order/search.js 12~14번째 줄)
const filteredOrders = orders.filter((item) => {
  const recipientName = item.shippingAddress.recipientName.toLowerCase() ?? "";
  const orderNumber = item.orderNumber.toLowerCase() ?? "";

// 수정 후
const filteredOrders = orders.filter((item) => {
  const recipientName = String(item.shippingAddress?.recipientName ?? "").toLowerCase();
  const orderNumber = String(item.orderNumber ?? "").toLowerCase();
```

**왜 이렇게 고치나요?**

코드 실행 순서를 그림으로 보면:

```
[수정 전] 에러가 나는 경우:
item.shippingAddress.recipientName.toLowerCase() ?? ""
│                    │                │             │
│                    │                ▼             │
│                    │         null.toLowerCase()   │
│                    │           → ❌ TypeError!     │
│                    │         (여기서 멈춤)         │
│                    │                              ▼
│                    │                        ?? 까지 도달 못함

[수정 후] 안전한 경우:
String(item.shippingAddress?.recipientName ?? "").toLowerCase()
│      │                  │               │    │            │
│      │                  ▼               │    │            │
│      │            undefined             │    │            │
│      │                  └─────── ?? ─── ▼    │            │
│      │                           = ""        │            │
│      ▼                                       │            │
│  String("")                                  │            │
│    = ""                                      │            │
│      └──────────────────────────── .toLowerCase() ──▶ ""  │
│                                                    ✅ 성공!│
```

핵심은 **null 체크를 먼저 하고, 그 다음에 메서드를 호출**하는 거예요.

`?.`(옵셔널 체이닝)도 추가했어요. `item.shippingAddress` 자체가 없을 수 있기 때문이에요:

```javascript
item.shippingAddress?.recipientName
//                  ↑ "shippingAddress가 있으면 recipientName을 가져오고, 없으면 undefined"
```

> **쉽게 말하면:** "냉장고에서 우유를 꺼내서 따르기"를 할 때, 냉장고에 우유가 있는지 **먼저 확인**하고 따라야 해요. 없는 우유를 따르려고 하면 에러(우유를 흘림)가 나겠죠!

**참고:** 같은 프로젝트의 `user/search.js`에서는 이미 올바른 패턴을 사용하고 있어요:

```javascript
// user/search.js — 이미 올바르게 작성됨! 👍
const userId = String(item.userId ?? "").toLowerCase();
```

이 패턴을 `order/search.js`에도 동일하게 적용하면 됩니다.

`admin/pages/components/user/deleteButtonEvent.js` 13번째 줄에서, 삭제 버튼을 클릭하면 확인 대화창 없이 바로 DELETE API가 호출됩니다.

```html
  async function handleLogin(e) {
    e.preventDefault();
    const email = document.getElementById("email").value.trim();
    const password = document.getElementById("password").value;
    const errorMsg = document.getElementById("errorMsg");
    const btn = document.getElementById("submitBtn");

    // 간단한 유효성 검사

    btn.disabled = true;

      const res = await fetch(

          method: "POST",
          headers: { "Content-Type": "application/json" },

      setTimeout(() => {
        window.location.href = "admin-dashboard.html";
      }, 500);
    } catch (error) {
      errorMsg.textContent = "이메일 또는 비밀번호가 올바르지 않습니다.";
      errorMsg.classList.remove("hidden");
```

## 3-6. API 함수 5개가 거의 똑같은 코드를 반복하고 있어요 (중요도: 중간)

### 문제

`admin/pages/API/` 폴더의 5개 파일(`orderListApi.js`, `orderDetailsApi.js`, `orderPatchApi.js`, `userListApi.js`, `userDeleteApi.js`)에서 토큰 확인, fetch 호출, 에러 처리 코드가 동일하게 반복됩니다.

### 해결 방법

**Step 1.** 새 파일 `admin/pages/API/fetchWithAuth.js`를 만드세요.

```javascript
// admin/pages/API/fetchWithAuth.js (새 파일)
const BASE_URL = "https://api.fullstackfamily.com/api/gentlelion/v1/admin";

export async function fetchWithAuth(endpoint, options = {}) {
  // 1. 토큰 확인
  const token = localStorage.getItem("token");
  if (!token) {
    console.error("토큰이 없습니다. 로그인 페이지로 이동합니다.");
    window.location.href = "/admin/pages/admin-login.html";
    return null;
  }

  try {
    // 2. API 호출
    const res = await fetch(`${BASE_URL}${endpoint}`, {
      ...options,
      headers: {
        Authorization: `Bearer ${token}`,
        ...options.headers,
      },
    });

    // 3. 응답 확인
    if (!res.ok) {
      throw new Error(`API 요청 실패: ${res.status} ${res.statusText}`);
    }

    return await res.json();
  } catch (error) {
    console.error(error);
    return null;
  }
}
```

**Step 2.** 각 API 파일을 수정하세요. 아래는 5개 파일 모두의 수정 전/후입니다.

**수정 전 vs 수정 후 비교:**

```

수정 후: 1개 공용 파일(30줄) + 5개 파일(각 5줄) = 약 55줄
```

## 3-8. 검색 이벤트 리스너가 페이지를 넘길 때마다 계속 쌓여요 (중요도: 중간)

### 문제

`admin/pages/components/order/search.js`와 `admin/pages/components/user/search.js`에서 `search()` 함수가 호출될 때마다 새 `input` 이벤트 리스너가 추가되지만, 이전 리스너가 제거되지 않습니다.

### 해결 방법

**Step 1.** `admin/pages/components/order/search.js` 전체를 수정하세요.

```javascript
// 수정 전 (admin/pages/components/order/search.js 전체)
export function search(orders, input, onSearch) {
  if (!input) return;

  input.addEventListener("input", () => {
    const keyword = input.value.trim().toLowerCase();

    if (!keyword) {
      onSearch(orders);
      return;
    }

    const filteredOrders = orders.filter((item) => {
      const recipientName = item.shippingAddress.recipientName.toLowerCase() ?? "";
      const orderNumber = item.orderNumber.toLowerCase() ?? "";

      return (
        recipientName.includes(keyword) ||
        orderNumber.includes(keyword)
      );
    });

    onSearch(filteredOrders);
  });
}

// 수정 후
let currentHandler = null;

export function search(orders, input, onSearch) {
  if (!input) return;

  // 이전 리스너 제거 (중복 방지)
  if (currentHandler) {
    input.removeEventListener("input", currentHandler);
  }

  // 새 리스너 생성
  currentHandler = () => {
    const keyword = input.value.trim().toLowerCase();

    if (!keyword) {
      onSearch(orders);
      return;
    }

    const filteredOrders = orders.filter((item) => {
      const recipientName = String(item.shippingAddress?.recipientName ?? "").toLowerCase();
      const orderNumber = String(item.orderNumber ?? "").toLowerCase();

      return (
        recipientName.includes(keyword) ||
        orderNumber.includes(keyword)
      );
    });

    onSearch(filteredOrders);
  };

  // 새 리스너 등록
  input.addEventListener("input", currentHandler);
}
```

**Step 2.** `admin/pages/components/user/search.js`도 같은 패턴으로 수정하세요.

```javascript
// 수정 후 (admin/pages/components/user/search.js 전체)
let currentHandler = null;

export function search(users, input, onSearch) {
  if (!input) return;

  // 이전 리스너 제거
  if (currentHandler) {
    input.removeEventListener("input", currentHandler);
  }

  // 새 리스너 생성
  currentHandler = () => {
    const keyword = input.value.trim().toLowerCase();

    if (!keyword) {
      onSearch(users);
      return;
    }

    const filteredUsers = users.filter((item) => {
      const userId = String(item.userId ?? "").toLowerCase();
      const userEmail = String(item.email ?? "").toLowerCase();

      return (
        userId.includes(keyword) ||
        userEmail.includes(keyword)
      );
    });

    onSearch(filteredUsers);
  };

  input.addEventListener("input", currentHandler);
}
```

**왜 이렇게 고치나요?**

```
[수정 전] 리스너 쌓임 문제:
1페이지 로드 → addEventListener(handler_1)
2페이지 로드 → addEventListener(handler_2)   ← handler_1도 여전히 있음!
3페이지 로드 → addEventListener(handler_3)   ← handler_1, handler_2도 있음!

검색창에 "홍"을 치면:
  handler_1 실행 → 1페이지 데이터로 검색 → 화면 업데이트
  handler_2 실행 → 2페이지 데이터로 검색 → 화면 또 업데이트!  ← 덮어씌워짐
  handler_3 실행 → 3페이지 데이터로 검색 → 화면 또 업데이트!  ← 결국 이것만 보임
  → 3개 함수가 모두 실행됨 (성능 낭비)

[수정 후] 항상 리스너 1개:
1페이지 로드 → addEventListener(handler_1)
2페이지 로드 → removeEventListener(handler_1), addEventListener(handler_2)
3페이지 로드 → removeEventListener(handler_2), addEventListener(handler_3)

검색창에 "홍"을 치면:
  handler_3만 실행 → 3페이지 데이터로 검색 → 화면 업데이트 ✅
```

핵심: `removeEventListener`를 하려면 **같은 함수 참조**가 필요해요. 그래서 `currentHandler` 변수에 함수를 저장해두고, 나중에 그걸 제거하는 거예요.

> **쉽게 말하면:** 새 친구 전화번호를 저장할 때, 이전에 잘못 저장한 번호를 먼저 삭제하고 새로 저장하는 거예요. 안 그러면 연락처에 같은 사람이 여러 개 생겨요.
